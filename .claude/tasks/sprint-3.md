# Sprint 3: Differentiators (Weeks 5-6)

## Goal
Ship features PB Vision doesn't have. These are our competitive moat.

## Tasks

### Task 3.1: Drill Mode
**Skills:** `.claude/skills/drill-mode/SKILL.md`
**Files:** New `src/analysis/drill_stats.py`, new `src/analysis/drill_detector.py`, modify `src/pipeline.py`, modify `modal_app.py`
**Acceptance:**
- [ ] User can select "drill" mode on upload
- [ ] Auto-detection: game vs drill (or manual selection)
- [ ] Drill stats: rep count, accuracy %, consistency score
- [ ] Supports: serve practice, dink rallies, 3rd shot drops
- [ ] Improvement tracking: first half vs second half accuracy
- [ ] Frontend: drill results view (separate from game view)
- [ ] API: `video_type: "drill"` parameter

### Task 3.2: Rally Scoring Support
**Skills:** `.claude/skills/score-reconstruction/SKILL.md`
**Files:** `src/analysis/score_tracker.py`, modify API + frontend
**Acceptance:**
- [ ] Scoring mode: "sideout" (default) or "rally"
- [ ] Rally scoring: every rally winner gets a point (no serving requirement)
- [ ] Games to 21, win by 2
- [ ] User can select scoring mode on upload
- [ ] Frontend: scoring mode toggle
- [ ] API: `scoring_mode: "sideout" | "rally"` parameter

### Task 3.3: Open Public API + Documentation
**Files:** Modify `modal_app.py`, new `docs/api.md`
**Acceptance:**
- [ ] API is publicly accessible (no auth for basic usage)
- [ ] Rate limiting: 10 requests/min per IP
- [ ] API documentation page (OpenAPI/Swagger auto-generated by FastAPI)
- [ ] Markdown API docs in `docs/api.md`
- [ ] Example curl commands for all endpoints
- [ ] Python client example code

### Task 3.4: Shareable Results Page
**Files:** New `frontend/app/results/[id]/page.tsx`, modify `modal_app.py`
**Acceptance:**
- [ ] Each analysis generates a unique ID
- [ ] Results stored persistently (Modal Volume or DB)
- [ ] Shareable URL: `yourdomain.com/results/{analysis_id}`
- [ ] OG meta tags for social sharing preview
- [ ] Results page loads without auth
- [ ] Expiry: results available for 30 days

## Dependencies
- Task 3.1 is the biggest — can be done by a dedicated sub-agent
- Task 3.2 extends score tracker from Sprint 1
- Task 3.3 is independent
- Task 3.4 needs persistent storage (Modal Volume or add a DB)
